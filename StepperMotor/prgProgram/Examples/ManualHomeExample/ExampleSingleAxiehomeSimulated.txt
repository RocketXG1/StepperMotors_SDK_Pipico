# ExampleSingleAxiehomeSimulated.cpp

## Finalidad del programa

Este ejemplo tiene la estructura general de `prgProject/main.cpp`, pero está enfocado en ejecutar HOME simulado para un solo eje.

La intención es que el archivo pueda copiarse o renombrarse como `main.cpp` cuando se quiera probar únicamente un eje, por ejemplo el eje Z, sin ejecutar HOME de todos los motores.

## Eje objetivo

El eje objetivo se define con:

```cpp
const char* targetHomeAxisName = "Z";
```

Si se quiere simular otro eje, cambiar el texto:

```cpp
const char* targetHomeAxisName = "X";
```

o:

```cpp
const char* targetHomeAxisName = "Y";
```

## Métodos utilizados

### `loadRecipeFromMicroSdOrFallback(recipe)`

Carga la receta desde microSD o usa la receta embebida de respaldo. Los nombres de los ejes y la bandera `homeSimulationEnabled` vienen de esta receta.

### `initializeMotionSystem(recipe, pulseEngine, motionSystem)`

Instancia los ejes usando el orden cargado desde la receta. El índice final del eje depende del orden en que aparezca en la receta.

### `motion.getAxisCount()`

Devuelve cuántos ejes están cargados.

### `motion.getAxisName(axisIndex)`

Devuelve el nombre del eje instanciado en ese índice. El programa usa este método para encontrar el índice real del eje deseado.

### `std::strcmp(...)`

Compara el nombre del eje actual con `targetHomeAxisName`.

Ejemplo:

```cpp
std::strcmp(motion.getAxisName(axisIndex), targetHomeAxisName) == 0
```

Si el resultado es `0`, significa que los nombres son iguales.

### `motion.init()`

Inicializa hardware de movimiento: PIO, ejes, pines DIR, ENABLE, límites y HOME según la configuración.

### `motion.printAxisMap()`

Imprime el mapa real de ejes instanciados:

```text
Index [0] = Axis X ...
Index [1] = Axis Y ...
Index [2] = Axis Z ...
```

Esto sirve para confirmar en consola cuál eje quedó en cada índice.

### `isHomeSimulationEnabled()`

Verifica que el eje encontrado esté configurado para HOME simulado desde la receta.

### `setSimulatedHomeActive(true)`

Marca el sensor HOME simulado del eje objetivo como activo. Esto equivale a decir:

```text
El eje objetivo ya tocó su sensor HOME.
```

### `motion.homeAxis(targetAxisIndex)`

Ejecuta HOME solo para el eje indicado. Como el sensor simulado ya fue activado, el controlador marca ese eje como `homed`, fija su posición en `homePositionMm` y no necesita mover físicamente el motor.

## Flujo del programa

1. Inicializa USB/stdio.
2. Carga la receta.
3. Crea el motor PIO de STEP.
4. Inicializa `MotionSystem`.
5. Obtiene `AdvanceMotionController`.
6. Busca el eje cuyo nombre coincida con `targetHomeAxisName`.
7. Si el eje no existe, detiene el sistema.
8. Inicializa el hardware con `motion.init()`.
9. Imprime el mapa de ejes con `motion.printAxisMap()`.
10. Verifica que el eje objetivo tenga `homeSimulationEnabled=true` o `1`.
11. Activa el sensor simulado con:

```cpp
motionSystem.axes[targetAxisIndex]->setSimulatedHomeActive(true);
```

12. Ejecuta HOME solo de ese eje con:

```cpp
motion.homeAxis(targetAxisIndex);
```

13. Si falla, ejecuta `emergencyStop()`.
14. Si termina bien, imprime mensaje de éxito y queda en ciclo infinito.

## Requisitos en la receta

Para el eje que se quiere simular:

```text
homeEnabled=true
homeSimulationEnabled=1
homePositionMm=0.0
```

Los otros ejes pueden tener `homeSimulationEnabled=0` si no se quieren probar.

## Uso recomendado

1. Elegir el eje modificando `targetHomeAxisName`.
2. Verificar en `axes.txt` que ese eje tenga `homeSimulationEnabled=1`.
3. Copiar o renombrar `ExampleSingleAxiehomeSimulated.cpp` como `main.cpp` si se quiere compilar como firmware principal.
4. Compilar y cargar el programa.
5. Revisar por consola que el eje fue encontrado y que HOME simulado terminó correctamente.
