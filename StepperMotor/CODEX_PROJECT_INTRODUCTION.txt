============================================================
1. OBJETIVO GENERAL DEL PROYECTO
============================================================

Este proyecto es un primer template base para controlar uno o varios motores paso a paso usando una Raspberry Pi Pico / RP2040 con Pico SDK C/C++.

El proyecto esta diseñado para controlar de 1 a 8 ejes. En el archivo main.cpp actual se instancian 3 ejes llamados X, Y y Z.

La arquitectura separa claramente:

- Configuracion fisica y mecanica de cada eje.
- Objeto individual por eje.
- Motor de pulsos STEP usando PIO.
- Controlador general de movimiento.
- Perfil trapezoidal de velocidad.
- Planificador de movimientos en cola.
- Suavizado geometrico de trayectoria para esquinas.
- HOME por eje.
- Orden de HOME configurable por nombre de eje.
- Validacion de nombres de eje.
- Validacion de pines STEP dentro del grupo PIO.
- Limites fisicos minimos/maximos por GPIO.
- Limites por software minimos/maximos en mm.

La finalidad es tener una plantilla inicial ordenada, entendible y extensible para que Codex pueda continuar el desarrollo sin perder el criterio de arquitectura ya definido.

------------------------------------------------------------
3.3 main.cpp
------------------------------------------------------------

Funcion:

Es el punto de entrada del firmware. Aqui se instancian y configuran los 3 ejes actuales: X, Y y Z.

Flujo general de main.cpp:

1. Inicializa stdio con stdio_init_all().
2. Espera 2 segundos para permitir abrir monitor serial.
3. Crea StepperPioPulseEngine.
4. Crea configuracion de eje X con StepperAxisConfig::createBeltDrive().
5. Asigna nombre obligatorio X con configX.setAxisName("X").
6. Configura HOME X, limites fisicos y limites por software.
7. Crea configuracion de eje Y con StepperAxisConfig::createBeltDrive().
8. Asigna nombre obligatorio Y.
9. Configura HOME Y, limites fisicos y limites por software.
10. Crea configuracion de eje Z con StepperAxisConfig::createLeadScrew().
11. Asigna nombre obligatorio Z.
12. Configura HOME Z, limites fisicos y limites por software.
13. Instancia AdvanceStepperAxis para X, Y y Z.
14. Opcionalmente permite activar simulacion de HOME.
15. Crea el arreglo AdvanceStepperAxis* axes[].
16. Instancia AdvanceMotionController motion(axes, 3, pulseEngine).
17. Valida motion.isValid().
18. Imprime mapa de ejes con motion.printAxisMap().
19. Configura orden de HOME por nombre con motion.setHomeOrderByName({"Z", "X", "Y"}).
20. Imprime el orden HOME con motion.printHomeOrder().
21. Inicializa el controlador motion.init().
22. Habilita drivers con axisX.enable(), axisY.enable(), axisZ.enable().
23. Ejecuta HOME con motion.homeAll().
24. Mueve a posicion segura despues de HOME.
25. Configura suavizado de trayectoria.
26. Carga movimientos en cola con queueLinearTo().
27. Prepara la cola con prepareQueuedMoves().
28. Ejecuta motion.update() continuamente en while(true).

Instanciacion principal:

StepperPioPulseEngine pulseEngine(
    pio0,
    0,
    2,
    10
);

Significado:

- pio0: bloque PIO usado.
- 0: maquina de estados PIO usada.
- 2: primer GPIO del grupo STEP.
- 10: intervalo minimo entre eventos STEP en microsegundos.

Grupo STEP:

- firstStepGpio = 2
- STEP_GROUP_PIN_COUNT = 8
- Pines disponibles para STEP: GPIO 2, 3, 4, 5, 6, 7, 8, 9.

Instancias de ejes:

AdvanceStepperAxis axisX(configX);
AdvanceStepperAxis axisY(configY);
AdvanceStepperAxis axisZ(configZ);

Arreglo de ejes:

AdvanceStepperAxis* axes[] = {
    &axisX,
    &axisY,
    &axisZ
};

Relacion interna:

- axes[0] = X
- axes[1] = Y
- axes[2] = Z

Aunque el HOME puede configurarse por nombre, el controlador sigue usando internamente el arreglo.

Orden HOME por nombre:

motion.setHomeOrderByName({"Z", "X", "Y"});

Esto internamente busca:

- "Z" -> axes[2]
- "X" -> axes[0]
- "Y" -> axes[1]

Y luego ejecuta:

- homeAxis(2)
- homeAxis(0)
- homeAxis(1)

HOME es secuencial, no paralelo.

4.1 StepperAxisConfig.h
------------------------------------------------------------

Funcion:

Define la estructura StepperAxisConfig, que almacena todos los parametros fisicos, electricos, mecanicos, de HOME, de limites y de nombre de un eje.

Tambien define el enum AxisMechanismType.

Enum:

AxisMechanismType:

- LEAD_SCREW: eje por husillo.
- BELT_DRIVE: eje por banda.

Variables principales de StepperAxisConfig:

1. PIN_NOT_USED

Constante usada para indicar que un pin no se usa.

2. AXIS_NAME_MAX_LENGTH

Tamanio maximo del nombre del eje.

3. axisName

Arreglo char que guarda el nombre del eje en mayusculas.

Reglas del nombre:

- Es obligatorio cambiarlo.
- Por defecto es UNNAMED.
- UNNAMED no se permite en el controlador.
- No puede repetirse entre ejes.
- setAxisName() convierte a mayusculas.

4. stepPin

GPIO usado para STEP del eje.

Debe pertenecer al grupo PIO controlado por StepperPioPulseEngine.

5. dirPin

GPIO usado para DIR del driver.

6. enablePin

GPIO usado para ENABLE del driver.

7. enableActiveLow

Indica si el driver se habilita con 0 logico.

8. directionInvert

Invierte la senial DIR si el motor se mueve al lado contrario.

9. motorStepsPerRev

Pasos completos por vuelta del motor. Ejemplo tipico: 200.

10. microsteps

Microstepping configurado en el driver. Ejemplo: 16.

11. mechanismType

Tipo de mecanismo: husillo o banda.

12. travelPerRevolutionMm

Distancia en mm que avanza el eje por cada vuelta del motor o salida mecanica.

Para banda:

travelPerRevolutionMm = beltPitchMm * pulleyTeeth

Para husillo:

travelPerRevolutionMm = screwPitchMm

13. gearRatio

Relacion mecanica. 1.0 significa transmision directa.

14. maxSpeedMmS

Velocidad maxima permitida para el eje en mm/s.

15. accelerationMmS2

Aceleracion maxima permitida para el eje en mm/s^2.

Variables HOME:

- homeEnabled: activa HOME para el eje.
- homePin: GPIO del sensor HOME.
- homeActiveLow: sensor activo en bajo.
- homeUsePullup: activa pull-up interno.
- homeDirectionPositive: direccion de busqueda HOME.
- homeSimulationEnabled: permite HOME simulado.
- homingSeekSpeedMmS: velocidad rapida de busqueda.
- homingLatchSpeedMmS: velocidad lenta de precision.
- homingBackoffMm: distancia de retroceso tras tocar sensor.
- maxHomingTravelMm: distancia maxima de busqueda HOME.
- homePositionMm: posicion logica asignada al terminar HOME.

Variables limites fisicos:

- minLimitEnabled
- minLimitPin
- minLimitActiveLow
- minLimitUsePullup
- maxLimitEnabled
- maxLimitPin
- maxLimitActiveLow
- maxLimitUsePullup

Variables limites por software:

- softLimitsEnabled
- softLimitMinMm
- softLimitMaxMm

Metodos de StepperAxisConfig:

setAxisName(const char* name)

Funcion:

- Copia el nombre al arreglo axisName.
- Convierte caracteres de a-z a A-Z.
- Rechaza nullptr.
- Regresa true si encontro al menos un caracter valido que no sea espacio/tab/salto de linea.

createBase()

Funcion:

- Crea una configuracion base con valores seguros por defecto.
- Asigna nombre UNNAMED.
- Inicializa pines como PIN_NOT_USED.
- Define valores default para motor, HOME, limites y velocidades.

Importante:

El nombre UNNAMED es intencionalmente invalido para forzar al usuario a asignar un nombre real.

createLeadScrew(...)

Funcion:

Crea una configuracion para eje por husillo.

Parametros principales:

- stepPin: pin STEP.
- dirPin: pin DIR.
- enablePin: pin ENABLE.
- enableActiveLow: define logica ENABLE.
- directionInvert: invierte direccion.
- motorStepsPerRev: pasos por vuelta del motor.
- microsteps: microstepping.
- screwPitchMm: avance del husillo por vuelta.
- gearRatio: relacion mecanica.
- maxSpeedMmS: velocidad maxima.
- accelerationMmS2: aceleracion maxima.

createBeltDrive(...)

Funcion:

Crea una configuracion para eje por banda.

Parametros adicionales:

- beltPitchMm: paso de la banda.
- pulleyTeeth: dientes de la polea.

Calcula:

travelPerRevolutionMm = beltPitchMm * pulleyTeeth

5.3 stepper.pio
------------------------------------------------------------

Programa PIO que genera pulsos STEP para 8 pines consecutivos.

Funcionamiento:

- Espera una mascara por FIFO.
- out pins, 8 pone en alto los pines indicados por la mascara.
- mov pins, null baja todos los pines a 0.
- Cada dato enviado por CPU representa un evento STEP simultaneo para uno o varios ejes.

Formato de mascara:

bit 0 = STEP eje con offset 0
bit 1 = STEP eje con offset 1
bit 2 = STEP eje con offset 2
...
bit 7 = STEP eje con offset 7

Ejemplo:

firstStepPin = 2

- GPIO 2 offset 0, mascara 0b00000001
- GPIO 3 offset 1, mascara 0b00000010
- GPIO 4 offset 2, mascara 0b00000100

Si se requiere step simultaneo X y Y:

mask = 0b00000011

Bloque C-SDK dentro de stepper.pio:

stepper_multi_program_init(...)

Configura:

- GPIOs STEP.
- Direccion de pines PIO.
- out pins.
- out shift.
- FIFO unido TX.
- divisor de clock.
- inicializacion y encendido de state machine.

============================================================
7. CARPETA src/srcMotion
============================================================

Contiene implementaciones de movimiento.

Archivos:

1. TrapezoidProfile.cpp
2. MotionPlanner.cpp
3. AdvanceMotionController.cpp

------------------------------------------------------------
7.1 TrapezoidProfile.cpp
------------------------------------------------------------

Implementa calculo de perfil trapezoidal/triangular.

Logica general:

1. Valida totalSteps, distanceMm, velocidad y aceleracion.
2. Limita entrySpeed y exitSpeed a rango valido.
3. Calcula distancia necesaria para acelerar hasta velocidad solicitada.
4. Calcula distancia necesaria para desacelerar desde velocidad solicitada.
5. Si ambas caben en la distancia total, usa perfil trapezoidal.
6. Si no caben, calcula velocidad pico alcanzable y usa perfil triangular.
7. Convierte distancias de aceleracion/desaceleracion a pasos relativos.

------------------------------------------------------------
7.2 MotionPlanner.cpp
------------------------------------------------------------

Implementa el planner.

Flujo de buildPlannedQueue():

1. Borra cola planeada.
2. Construye lista de puntos: posicion inicial + targets crudos.
3. Inicializa arrays de puntos de entrada/salida de curvas.
4. Si blendEnabled, rawCount >= 2 y axisCount >= 2, analiza esquinas.
5. Para cada esquina:
   - Calcula vector de entrada.
   - Calcula vector de salida.
   - Calcula dot product.
   - Omite si es casi linea recta.
   - Omite si es cambio casi 180 grados.
   - Calcula trimDistance.
   - Calcula punto antes y despues de esquina.
   - Marca esquina con blend.
6. Recorre movimientos crudos.
7. Agrega linea hasta punto antes de esquina si aplica.
8. Si hay blend, genera segmentos Bezier hacia punto despues de esquina.
9. Asigna velocidades entry/exit.

Importante:

El suavizado se aplica solo al usar:

- queueLinearTo()
- prepareQueuedMoves()
- update()

No se aplica al usar moveLinearTo().

------------------------------------------------------------
7.3 AdvanceMotionController.cpp
------------------------------------------------------------

Implementa el controlador principal.

Flujo del constructor:

1. Inicializa variables.
2. Limpia arrays.
3. loadAxes().
4. validateAxesStepPins().
5. validateAxisNames().
6. Configura MotionPlanner.

Validaciones importantes:

validateAxesStepPins()

- Verifica que cada stepPin pertenezca al grupo PIO.
- Verifica que no haya offsets STEP repetidos.

validateAxisNames()

- Rechaza nombres nullptr.
- Rechaza nombres vacios.
- Rechaza UNNAMED.
- Rechaza duplicados.

setHomeOrderByName()

- Recibe nombres como "Z", "X", "Y".
- Normaliza a mayusculas.
- Busca el indice interno de cada nombre.
- Evita repetidos.
- Llena homeOrder con indices.

homeAll()

- Requiere homeOrderConfigured=true.
- Ejecuta HOME en orden.
- Es secuencial y bloqueante.
- Si falla un eje, devuelve false.

homeAxis()

Secuencia de HOME:

1. Valida controlador, eje, busy y HOME enabled.
2. Habilita eje.
3. Si HOME simulado esta activo y sensor simulado activo, asigna homePositionMm y termina.
4. Determina direccion HOME.
5. Calcula delay de busqueda rapida.
6. Calcula delay de entrada lenta.
7. Convierte maxHomingTravelMm a pasos.
8. Busca sensor HOME rapido.
9. Si excede distancia maxima, emergencyStop().
10. Retrocede homingBackoffMm en sentido contrario.
11. Espera 100 ms.
12. Entra lento hasta sensor.
13. Asigna posicion logica HOME.
14. Marca eje como homed.

executeBresenhamStep()

Funcion principal de sincronizacion de ejes:

1. Si termino dominantSteps, libera busy.
2. Inicializa stepMask.
3. Recorre ejes.
4. Acumula bresenhamError.
5. Si un eje debe dar paso, verifica limite fisico.
6. Calcula pinOffset del STEP.
7. Agrega bit a stepMask.
8. Actualiza positionDelta.
9. Envia stepMask al PIO.
10. Actualiza posicion en pasos.
11. Incrementa dominantStepCounter.

update()

Debe llamarse constantemente en loop.

Si no hay movimiento activo, intenta cargar siguiente movimiento planeado.

Si hay movimiento activo y ya llego nextStepTime:

- Ejecuta paso Bresenham.
- Calcula nuevo delay.
- Actualiza nextStepTime.

============================================================
9. FLUJO DE MOVIMIENTO DIRECTO
============================================================

Uso:

motion.moveLinearTo({100.0f, 50.0f, 10.0f}, 40.0f);

while (motion.isBusy()) {
    motion.update();
}

Flujo interno:

1. moveLinearTo() valida que no este busy.
2. buildLinearBlockTo() crea un MotionBlock.
3. buildLinearBlockToArray() calcula deltas, pasos, direcciones y limites.
4. startMotionBlock() calcula perfil trapezoidal.
5. update() ejecuta Bresenham.
6. pulseEngine manda mascaras STEP.

Importante:

moveLinearTo() NO usa MotionPlanner ni suavizado geometrico. Es un movimiento directo.

============================================================
10. FLUJO DE MOVIMIENTO EN COLA CON SUAVIZADO
============================================================

Uso:

motion.configurePathSmoothing(true, 2.0f, 0.5f, 12, 0.70f);

motion.clearMotionQueue();
motion.queueLinearTo({100.0f, 0.0f, 10.0f}, 40.0f);
motion.queueLinearTo({100.0f, 100.0f, 10.0f}, 40.0f);
motion.queueLinearTo({200.0f, 100.0f, 10.0f}, 40.0f);
motion.prepareQueuedMoves();

while (motion.isBusy() || motion.getPlannedMoveCount() > 0) {
    motion.update();
}

Flujo interno:

1. queueLinearTo() guarda movimientos crudos.
2. prepareQueuedMoves() fija posicion inicial actual.
3. MotionPlanner::buildPlannedQueue() analiza esquinas.
4. Si suavizado activo, genera segmentos Bezier.
5. update() carga movimientos planeados uno por uno.
6. startPlannedLinearMove() convierte cada planned move en MotionBlock.
7. executeBresenhamStep() ejecuta pulsos sincronizados.

============================================================
11. HOME POR NOMBRE
============================================================

La forma recomendada es:

motion.setHomeOrderByName({"Z", "X", "Y"});

Beneficios:

- No depende visualmente de indices.
- Evita confundir axes[0], axes[1], axes[2].
- Permite cambiar orden interno de arreglo sin reescribir HOME manualmente, siempre que los nombres esten correctos.

Reglas:

- Los nombres se normalizan a mayusculas.
- No se permiten nombres duplicados.
- No se permite UNNAMED.
- Si un nombre no existe, setHomeOrderByName() retorna false.

============================================================
12. VALIDACIONES IMPLEMENTADAS
============================================================

Validacion de ejes:

- numberOfAxes debe estar entre 1 y 8.
- axisList no puede ser nullptr.
- Ningun axisList[i] puede ser nullptr.

Validacion de pines STEP:

- Cada stepPin debe pertenecer al grupo PIO.
- No debe repetirse un stepPin/offset.

Validacion de nombres:

- No nullptr.
- No vacio.
- No UNNAMED.
- No duplicado.
- Comparacion sensible al texto ya normalizado a mayusculas.

Validacion de movimientos:

- Cantidad de coordenadas debe coincidir con axisCount.
- Velocidad solicitada debe ser mayor a 0.
- Target debe estar dentro de soft limits si estan activos.
- El movimiento debe tener distancia mayor a 0.
- El perfil trapezoidal debe ser valido.

Validacion de limites fisicos:

Antes de cada paso:

- Si la direccion es positiva y maxLimit esta activo, se bloquea.
- Si la direccion es negativa y minLimit esta activo, se bloquea.
- Si se bloquea, se ejecuta emergencyStop().

============================================================
13. COMPORTAMIENTO DE emergencyStop()
============================================================

emergencyStop() hace:

1. stop()
2. Limpia movimiento actual.
3. Limpia cola del planner.
4. Deshabilita todos los ejes.

En main.cpp actual, tras error grave se usa:

while (true) {
    tight_loop_contents();
}

Esto deja el firmware detenido intencionalmente.

Para una version futura, se recomienda reemplazar ciclos infinitos por una maquina de estados de falla con boton fisico o comando serial para reintentar HOME.

- Los STEP deben estar consecutivos.
- DIR, ENABLE, HOME y LIMITS pueden estar en otros GPIO.

18. LIMITACIONES ACTUALES CONOCIDAS
============================================================

1. HOME es bloqueante.

homeAxis() usa bucles while y stepAxisOnceBlocking(). Mientras hace HOME de un eje, no ejecuta movimiento paralelo.

2. homeAll() es secuencial.

Primero termina un eje, luego el siguiente.

3. No hay maquina de estados de fault recuperable.

En main.cpp actual, errores graves detienen el programa en while(true).

4. No hay comunicacion externa todavia.

El proyecto esta listo para agregarse serial, USB, TCP/IP u otro protocolo.

5. No hay persistencia de configuracion.

Los parametros se definen en main.cpp.

6. No hay interpolacion avanzada tipo GRBL.

El template implementa movimiento lineal sincronizado y suavizado basico por Bezier, no un planner industrial completo.

20. RESUMEN FINAL PARA CODEX
============================================================

Este proyecto es un template modular de control de movimiento para RP2040.

La idea central es:

- main.cpp configura e instancia.
- StepperAxisConfig describe cada eje.
- AdvanceStepperAxis opera cada eje individual.
- StepperPioPulseEngine genera los pulsos STEP por PIO.
- AdvanceMotionController coordina los ejes.
- MotionBlock representa un movimiento ejecutable.
- TrapezoidProfile calcula aceleracion/desaceleracion.
- MotionPlanner prepara colas y suavizado.
- stepper.pio ejecuta pulsos simultaneos por mascara.

El orden de HOME debe configurarse preferentemente con nombres:

motion.setHomeOrderByName({"Z", "X", "Y"});

Los movimientos simples deben usar:

motion.moveLinearTo(...);

Las trayectorias con multiples puntos y suavizado deben usar:

motion.queueLinearTo(...);
motion.prepareQueuedMoves();
motion.update();

Este documento debe servir como contexto inicial para Codex antes de modificar, extender, depurar o reorganizar el template.
